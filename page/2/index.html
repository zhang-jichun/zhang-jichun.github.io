2.1 最长回文子串

abcd 子串 ab,bc… 子序列 ac,bd…

  1. 暴力解法-相向型双指针-时间复杂度on3-不合格
  2. 马拉车算法on,并不是面试官需要的算法，面试并不考记忆。

2.2 on3代码实现差异

  1. 注意写题目要异常检测。
  2. 不要用单个字母变量。
  3. 加空格。逻辑块用空行分开。
  4. 缩进不要超过3层。

2.3 基于中心线枚举的算法

  1. on2时间复杂度。n个奇数长度中心点，n-1个偶数长度中心点。双指针L向左走R向右走。
  2. 避免重复代码，避免全局变量，避免递归。
  3. python作为面试语言可以缩短时间。

2.4 动态规划法

2.5 面试评价标准

  1. 不一定非要最优解，要贴近面试。
  2. BugFree
  3. CodingStyle
  4. 逻辑易懂
  5. 没有冗余代码
  6. 边界检测与异常处理
  7. 九章solution可查看答案
  8. noHire weakHire Hire strongHire

2.6 十二招快速提高

  1. codingQuality助力bugFree
  2. 二元运算符两边加空格，单元运算符不加空格
  3. 各种扩号和for，if之间加空格
  4. 逗号分号后面加空格
  5. 空行分割不同逻辑块
  6. 变量名用1-2单词
  7. 确保一个函数内部不超过3层缩进
  8. 包装子函数减少缩进
  9. 多用continue少用if，少用else
    ```
    for…
     if...   
     做一些处理
     做一些处理
    for…
     if not ...
     continue
 做一些处理
 做一些处理
    ```hes
  10. 对入口参数进行异常检测
  11. 确保下标不越界，确保对象不是空
  12. 不用全局变量

九章算法-01-高效刷题

发表于 2022-01-25 更新于 2022-01-31 分类于 九章算法
  • 有的题不要刷，有的题目可以多刷。
  • 缩进与循环尽量不要超过三层，若多层可以采用子函数，代码缩进越多bug越多。
  • 进步最快的是代码被更高级的员工code review。
  • magicNumber(定义wall=1,water=2)
  • 缺乏项目经验：耦合度高，重复代码，全局变量。
  • 解耦合：整体逻辑用子函数表达，子函数实现具体步骤。
  • 好的代码不需要注释。更清晰的变量名，更易懂的逻辑。
  • 子函数化，避免全局变量可以降低bug.
  • 逻辑思维能力，代码质量，代码风格，异常检测。
  • 面试是一个试用的过程，不是考试的过程。
  • 不要闷头写，先沟通清除再写，写完了再解释。
  • 不会刻意要提示，但先自己努力一下。
  • 会就会，不会就是不会，坦诚很重要。
  • 一个Django项目课，可以咨询。
  • 面试语言首推python,其次java,不要用C++。
  • lintcode 算法能力，bugFree能力，题量。
  • lintcodeCat帮助评估算法能力。
  • 判断算法考不考，带名字的算法都不考。
  • 参考考察表v4.1
  • 熟悉每一种常用的数据结构。
  • 互联网公司考systemDesign多，软件公司考OOD
  • OOD有5章节10课时的课程。
  • BehaviorQuestion也要有准备，主要准备与面试公司的联系。
